**ISO/IEC 27001:2022** specifies requirements for an Information Security
Management System (ISMS). Its Annex A lists 93 controls grouped into four
themes — Organizational, People, Physical, and Technological — that an
organization selects from based on a risk assessment. The SpecFleet ISO 27001
starter pack maps a subset of Annex A controls to technical hooks an AI
agent can enforce at runtime.

## Coverage matrix

| Control | Control name                                       | SpecFleet hook                                            | Coverage notes                                                              |
|---------|----------------------------------------------------|-----------------------------------------------------|-----------------------------------------------------------------------------|
| A.5.15  | Access control                                     | `egress-allowlist`, `ip-guard`                      | Network-layer access control via allow list; data-layer via DLP patterns.   |
| A.5.23  | Information security for use of cloud services     | `egress-allowlist`                                  | Restricts agents to vetted cloud endpoints (Azure, GitHub, registries).     |
| A.5.33  | Protection of records                              | `audit-hashchain`                                   | Hash-chain provides integrity of session records.                           |
| A.8.2   | Privileged access rights                           | `pre-commit-scan`, `secret-redaction`               | Pre-commit scan catches privileged credential leaks before they ship.       |
| A.8.12  | Data leakage prevention                            | `ip-guard`, `secret-redaction`, `egress-allowlist`  | DLP patterns + redaction + egress control compose into a layered defense.   |
| A.8.15  | Logging                                            | `audit-hashchain`                                   | Tamper-evident logs with stable, replayable record IDs.                     |
| A.8.28  | Secure coding                                      | `pre-commit-scan`                                   | Pre-commit scan blocks unsafe patterns (eval, telnet, hard-coded secrets).  |

## v0.5 pack behavior

In v0.5, `specfleet init --with-pack iso27001` populated an ISO 27001 policy
pack and registered selected Annex A controls in `.specfleet/instruction.md`.
In v0.6 there is no pack installer; keep the controls in the constitution and
verify evidence with the checklist phase.

## Disclaimer

Starter pack only — tune to your compliance program. ISO 27001 certification
also requires a documented ISMS, risk treatment plan, internal audit, and
management review — none of which are produced by SpecFleet.
